# Code of Conduct

## Our kitchen, our standards

AICard is a kitchen. A kitchen works when everyone respects the space, the tools, and each other. This code of conduct sets the expectations for everyone who contributes — whether you write code by hand, use AI tools, or contribute in any other way.

## Expected behavior

- Be respectful, constructive, and patient
- Use the project's [domain language](docs/AICard_Domain_Language.md) — it exists so we can communicate clearly
- Respect the [scope boundaries](CONTRIBUTING.md) — build what's needed, not what's interesting
- Give and receive feedback gracefully
- Focus on what is best for the project's users: Maria, Jun, and Sam

## Unacceptable behavior

- Harassment, discrimination, or personal attacks of any kind
- Dismissive or demeaning language toward other contributors
- Submitting bulk, untested contributions without regard for quality
- Misrepresenting AI-generated work as entirely hand-written when asked about it
- Publishing others' private information without consent

## AI-assisted contributions

AI-assisted work is welcome and subject to the same quality standards as any other contribution. We judge the output, not the tool. See the [AI contribution policy](CONTRIBUTING.md#contributing-with-ai-assistance) in CONTRIBUTING.md for disclosure expectations.

## Scope

This code of conduct applies to all project spaces: issues, pull requests, discussions, and any other communication channel associated with AICard.

## Enforcement

Instances of unacceptable behavior may be reported by contacting the project maintainers. All complaints will be reviewed and investigated and will result in a response that is deemed necessary and appropriate to the circumstances.
